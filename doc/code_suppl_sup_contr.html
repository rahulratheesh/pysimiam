class controllers.pid_controller.PIDController(params)[source]¶

The PID controller is a general-purpose controller that steers the robot to a certain heading direction. The heading is recalculated on every execution.

execute(state, dt)[source]¶

Calculate errors and steer the robot

get_heading(state)[source]¶

Get the direction in which the controller wants to move the robot as a vector in the robot’s frame of reference.

Returns:a numpy array [x, y, z] with z = 1.
get_heading_angle(state)[source]¶

Return the heading as an angle in the robot’s frame of reference.

restart()[source]¶

Set the integral and differential errors to zero

set_parameters(params)[source]¶

Set PID values

The params structure is expected to have in the gains field three parameters for the PID gains.

Parameters:
  • params.gains.kp (float) – Proportional gain
  • params.gains.ki (float) – Integral gain
  • params.gains.kd (float) – Differential gain
class controllers.gotogoal.GoToGoal(params)[source]¶

Go-to-goal steers the robot to a predefined position in the world.

get_heading_angle(state)[source]¶

class controllers.avoidobstacles.AvoidObstacles(params)[source]¶

Avoid obstacles is an example controller that checks the sensors for any readings, constructs ‘obstacle’ vectors and directs the robot in the direction of their weighted sum.

get_heading(state)[source]¶

Get the direction away from the obstacles as a vector.

set_parameters(params)[source]¶

Set PID values and sensor poses.

The params structure is expected to have sensor poses in the robot’s reference frame as params.sensor_poses.
